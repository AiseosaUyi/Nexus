# UX Writing Skill

This skill ensures that Nexus feels intuitive and friendly through language. Use it whenever UI copy or user experience text is mentioned.

## UX Writing Core Principles
- **Be Concise**: Use as few words as possible without losing meaning.
- **Be Helpful**: Provide "what to do next" instructions in every error message.
- **Be Consistent**: Maintain a unified tone across all modals, buttons, and settings.

## UX Workflows
1. **CTA Audit**: Review button labels (e.g., "Submit" vs "Get Started") for conversion impact.
2. **Empty State Design**: Write encouraging copy for when nodes or blocks are empty.
3. **Error Remediation**: Rewrite "Something went wrong" into actionable, human-readable instructions.

## Key Instructions
- **Draft Microcopy**: Create text for tooltips or help-icons.
- **Tone Check**: Ensure all UI elements are "Encouraging" and "Simple".
- **CTA Builder**: Draft 3 versions of a call-to-action button based on user goals.
- **Onboarding Copy**: Write the step-by-step instructions for getting started.
