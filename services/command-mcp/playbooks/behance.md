# Behance — playbook (kind: both)

Behance is portfolio + discovery. Reach-outs come via **project comments**, **appreciations**, and
**messages**. Being featured in a gallery drives inbound, so posting well matters.

## Read (each run)
- Notifications bell → new comments, appreciations, follows, messages.
- Messages inbox → collaboration/job asks. Run every message through `_SCAM-FILTER.md`
  (Behance is a known target for the "test this link" scam).

## Draft
- Comment replies: short, warm, specific (`cc_capture_opportunity` type=comment).
- Message/job asks: proposal-style, ask for scope + budget if missing.

## What a strong Behance post looks like (for scoring / content engine)
- A full **project case study**, not a single image: cover → problem → process → final shots.
- Cover thumbnail is everything — high contrast, one clear hero, readable at small size.
- 1600px+ wide images, consistent margins, short captions between sections.
- Tags filled, tools listed, "Available for work" toggle ON.
- Post when your audience is active; multi-image projects outperform singles.

## Health inputs
- Profile completeness, "available for work" on, projects in last 30-60 days, response speed,
  appreciations/views trend. Top fix is usually: publish one new full case study.

## Execute (after approval)
- Comment reply: open the project → reply box → paste approved text → post → `cc_mark_sent`.
- New project: this is a design task — Aise provides/points to the exported frames; you assemble
  the project in Behance's editor per the approved layout, then publish → `cc_mark_posted` + URL.
