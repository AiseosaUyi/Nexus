# Upwork — playbook (kind: inbound)

Upwork is proposals + client messages. Money platform. Connects cost money, so **quality over
volume** — only bid where fit ≥ 60.

## Read (each run)
- Messages tab → client replies, interview invites, offers.
- My Jobs → active contracts needing a nudge.
- (Optional) Best-matches / saved searches → new jobs worth a proposal.

## Draft
- Proposals: open with the client's specific problem, one concrete idea, relevant sample, then
  scope + timeline. Never a generic template. Capture as type=job with the job text in raw_text.
- Client messages: prompt, specific, move toward scoping a call or milestone.
- Score fit hard (budget, clarity, reviews, payment-verified). Skip vague $5 jobs.

## What good looks like on Upwork
- Profile: sharp title, specialized (not "I do everything"), portfolio pieces with outcomes,
  100% Job Success protected by only taking fits.
- Proposal: <150 words, no fluff, answers their screening questions, one link that proves it.

## Health inputs
- Response time (Upwork weights this heavily), proposal→reply rate, profile completeness,
  availability set, recent activity. Top fix is usually: tighten the profile title/specialization.

## Execute (after approval)
- Proposal: open the job → Submit a Proposal → paste approved text, set rate/connects Aise
  approved → submit → `cc_mark_sent`.
- Message: open thread → paste → send → `cc_mark_sent`.
- Never accept an offer or change rates without Aise doing it. Financial actions are Aise's.
