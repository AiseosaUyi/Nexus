# Dribbble — playbook (kind: both)

Dribbble is shots + "hire me." Inbound comes via **comments**, **messages/hire inquiries**, and
Dribbble's job board (if on the pro plan). **High scam target** — this is where Aise got the
fake-recruiter/"test this link" scam. Be strict.

## Read (each run)
- Notifications → comments, likes, follows.
- Messages / "Get Hired" inquiries → run EVERY one through `_SCAM-FILTER.md`.
  Fake recruiters here love: shortened links, "quick test task," off-platform chat, crypto pay.

## Draft
- Legit hire inquiry: proposal, confirm scope + budget, keep it on-platform.
- Comments: short and specific.
- Anything asking to click/test a link or move to Telegram → capture, let it quarantine, no reply.

## What a strong Dribbble shot looks like
- Single, tight, beautiful frame — one idea, portfolio-grade. 1600×1200 (or 4:3), retina-crisp.
- A real UI moment or a bold visual, not a full case study (that's Behance).
- Clean background, subtle shadow/mock, on-brand color.
- Title + a 1-2 line description + tags. "Available for work" ON.
- Attach a couple of extra frames in the shot for depth.

## Health inputs
- Availability badge on, recent shots (last 2-3 weeks), followers/likes trend, response speed.
  Top fix is usually: post 1-2 fresh shots this week.

## Execute (after approval)
- Reply/inquiry: open thread → paste approved text → send → `cc_mark_sent`.
- New shot: Aise exports the frame; you upload, set title/tags/availability per approved copy →
  publish → `cc_mark_posted` + URL.
